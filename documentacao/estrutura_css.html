Documentação - Estrutura CSS Modular

Visão Geral

Esta documentação descreve a estrutura modular CSS proposta para o projeto, organizando os estilos em arquivos lógicos e reutilizáveis.

Por que modularizar?

  • Organização: Cada tipo de estilo fica em seu próprio arquivo
  • Manutenibilidade: Fácil de encontrar e editar estilos específicos
  • Reusabilidade: Componentes podem ser usados em diferentes páginas
  • Performance: Pode carregar apenas os CSS necessários para cada página
  • Escalabilidade: Fácil de adicionar novos componentes ou páginas
  • Colaboração: Múltiplos devs podem trabalhar em arquivos diferentes sem conflitos
  • Legibilidade: Arquivos menores e mais focados são mais fáceis de entender

Estrutura de Arquivos

css/ ├── base/ │ ├── _variables.css /* Variáveis de cores e fontes */ │ ├── _reset.css /* Reset básico */ │ └── _typography.css /* Estilos de tipografia */ ├── components/ │ ├── _buttons.css /* Estilos de botões */ │ ├── _cards.css /* Estilos de cards */ │ ├── _forms.css /* Estilos de formulários */ │ └── _hero.css /* Estilos do hero section */ ├── layout/ │ ├── _header.css /* Estilos do cabeçalho */ │ ├── _footer.css /* Estilos do rodapé */ │ └── _grid.css /* Sistema de grid */ ├── pages/ │ ├── _home.css /* Estilos específicos da home */ │ └── _features.css /* Estilos da seção de features */ ├── themes/ │ └── _default.css /* Tema padrão */ ├── utils/ │ ├── _animations.css /* Animações */ │ └── _utilities.css /* Classes utilitárias */ └── main.css /* Arquivo principal que importa todos os outros */

Arquivos Principais

main.css /css/main.css

Arquivo principal que importa todos os outros módulos CSS. Funciona como ponto de entrada único para os estilos.

/* Base */
@import "base/variables";
@import "base/reset";
@import "base/typography";

/* Components */
@import "components/buttons";
@import "components/cards";
@import "components/forms";
@import "components/hero";

/* Layout */
@import "layout/header";
@import "layout/footer";
@import "layout/grid";

/* Pages */
@import "pages/home";
@import "pages/features";

/* Themes */
@import "themes/default";

/* Utils */
@import "utils/animations";
@import "utils/utilities";
_variables.css /css/base/_variables.css

Contém todas as variáveis CSS (cores, espaçamentos, etc.) usadas no projeto.

:root {
    --azul-tecnico: #0A66E4;
    --azul-escuro: #003366;

    --cinza-tecnico: #333333;

}

Implementação

Pré-requisitos

Para usar esta estrutura, você precisará de:

  • Um pré-processador CSS como Sass ou Less para lidar com os imports
  • Ou um bundler como Webpack, Parcel ou Vite

Passos para implementação

  1. Crie a estrutura de pastas conforme descrito acima
  2. Divida o CSS existente nos arquivos correspondentes
  3. Configure seu pré-processador ou bundler para compilar o main.css
  4. No HTML, link apenas o arquivo main.css compilado

Dicas para manutenção

  • Mantenha os arquivos focados em um único propósito
  • Nomeie os arquivos de forma clara e consistente
  • Adicione comentários explicando o propósito de cada seção
  • Atualize esta documentação quando adicionar novos módulos

Próximos Passos

Esta estrutura pode ser ainda mais aprimorada com:

  • Variáveis para espaçamentos e breakpoints
  • Documentação de estilo para componentes
  • Um sistema de design mais robusto
  • Integração com JavaScript para carregamento sob demanda
